# ASSET_REQUIREMENTS.md

# Kelza Asset Requirements

> "A premium experience is never created by code alone."

---

# Purpose

This document defines every visual asset required for the Kelza Experience Platform.

No contributor should assume placeholder boxes are acceptable.

Every major section should contain premium visual assets.

The frontend should look like a finished product, not a wireframe.

Whenever possible, assets should be original and created specifically for Kelza.

---

# Asset Philosophy

Every asset should communicate one of the following:

• Human connection

• Community

• Trust

• Discovery

• Technology

• Intelligence

• Creativity

• Opportunity

• Growth

• Hope

Assets should never exist only because they "look cool."

Every illustration, banner, mockup, icon, animation and background should strengthen the story.

# FINAL RULE

Never ship a section with missing visuals.

If a section exists,

it deserves premium assets.

If premium assets are unavailable,

create them.

Do not rely on placeholders.

The visual experience should communicate the same level of craftsmanship as the engineering.

Every asset should make someone believe that Kelza is a product worth remembering.
